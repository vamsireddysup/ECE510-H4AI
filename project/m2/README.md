# Milestone 2 -- QK^T Accelerator Chiplet

## Overview

This milestone implements the RTL compute core and interface module for
a custom chiplet accelerating QK^T matrix multiplication in transformer
self-attention. All simulations use Verilator 5.041 on Ubuntu 24.04.

---

## Repository structure

```
project/m2/
├── rtl/
│   ├── compute_core.sv   -- top-level systolic array (4x4 for M2, 8x8 target)
│   └── interface.sv      -- AXI4-Lite control interface
├── tb/
│   ├── tb_compute_core.cpp -- Verilator C++ testbench for systolic array
│   └── tb_interface.cpp    -- Verilator C++ testbench for AXI4-Lite
├── sim/
│   ├── compute_core_run.log -- simulation transcript showing 16/16 PASS
│   ├── interface_run.log    -- simulation transcript showing 2/2 PASS
│   └── waveform.png         -- GTKWave waveform from systolic array run
├── precision.md             -- FP32 precision choice and error analysis
└── README.md                -- this file
```

Additional HDL source files (required by compute_core.sv and interface.sv):
```
project/m2/hdl/src/
├── fp32_mul.sv          -- synthesizable FP32 multiplier (3-stage pipeline)
├── fp32_add.sv          -- synthesizable FP32 adder (1-cycle)
├── pe.sv                -- FP32 processing element with sequential accumulator
├── systolic_array.sv    -- parameterized NxN systolic array
├── axi4_lite_ctrl.sv    -- AXI4-Lite slave controller
└── axi4_stream.sv       -- AXI4-Stream data interface
```

## How to reproduce simulations

### Prerequisites

```
Verilator 5.041  (verilator --version)
g++ 12 or later  (g++ --version)
GTKWave 3.3.116  (gtkwave --version)  -- for waveform viewing only
```

### Clone and setup

```bash
git clone https://github.com/vamsireddysup/ECE510-H4AI.git
cd ECE510-H4AI
```

### Run compute core testbench (systolic array 16/16 QK^T verify)

```bash
cd project/m2/hdl/tb
make array
```

Expected output:
```
Results: 16/16 elements correct
ALL PASS -- QK^T output matches golden reference
```

### Run interface testbench (AXI4-Lite write + read transaction)

```bash
cd project/m2/hdl/tb
make axi
```

Expected output:
```
Results: 2/2 transactions passed
ALL PASS -- AXI4-Lite interface verified
```

### Run all testbenches

```bash
cd project/m2/hdl/tb
make pe && make array && make axi && make stream
```

### Run with debug traces

```bash
make pe_debug      -- FP32 PE internal pipeline traces
make array_debug   -- systolic array with PE traces
make axi_debug     -- AXI4-Lite transaction traces
make stream_debug  -- AXI4-Stream transaction traces
```

### View waveform

After running any testbench, open the generated VCD:

```bash
gtkwave project/m2/hdl/tb/systolic_wave.vcd
```

## Deviations from M1 plan

**Interface:** M1 selected AXI4-Lite (control) + AXI4-Stream (data).
Both are implemented as specified. No changes.

**Precision:** M1 documented FP32 as the target precision. FP32 is
implemented throughout using synthesizable fp32_mul.sv and fp32_add.sv
modules. No changes.

**Array size:** The systolic array is parameterized for NxN. M2
testbenches use SIZE=4 for a 4x4 array matching the golden reference
matrices. The target 8x8 array for 32 GFLOP/s is the M3 synthesis target.
